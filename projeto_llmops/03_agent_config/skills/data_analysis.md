# 📊 SKILL — DATA ANALYSIS (Análise Estruturada de Dados)

---

## 🎯 1. OBJETIVO DA SKILL

A skill de data_analysis é responsável por interpretar dados, gerar insights confiáveis e suportar tomada de decisão baseada em evidência.

Seu objetivo é:

- transformar dados brutos em informação útil
- gerar insights relevantes
- garantir fidelidade e consistência dos dados
- suportar decisões de negócio
- preparar dados para visualização ou automação

---

## 🧠 2. QUANDO UTILIZAR

Esta skill deve ser ativada quando houver:

- análise de dados (Excel, pandas, SQL)
- cálculo de indicadores (KPIs)
- interpretação de resultados
- identificação de padrões ou tendências
- necessidade de explicar comportamento dos dados

---

## 📊 3. PAPEL NO SISTEMA

A skill de data_analysis pode atuar como:

---

### ✅ Skill principal

Quando o objetivo for:

- analisar dados
- gerar insights
- interpretar métricas
- explicar resultados

---

### ✅ Skill de suporte

Combinada com:

- reasoning → estruturar interpretação
- code_generation → implementar análise em código
- validation → garantir consistência dos dados

---

## ⚙️ 4. LÓGICA DE EXECUÇÃO

A execução deve seguir obrigatoriamente:

---

### 1. Entendimento do contexto

- identificar o objetivo da análise
- entender o problema de negócio
- definir o que precisa ser respondido

---

### 2. Compreensão dos dados

- identificar estrutura dos dados
  (colunas, tipos, granularidade)
- validar qualidade dos dados
- identificar possíveis inconsistências

---

### 3. Definição de métricas

- identificar KPIs relevantes
- definir cálculos necessários
- alinhar com objetivo da análise

---

### 4. Análise dos dados

- aplicar agregações (soma, média, contagem)
- identificar padrões e tendências
- comparar períodos, grupos ou categorias

---

### 5. Geração de insights

- interpretar resultados obtidos
- identificar causas possíveis
- relacionar dados com contexto de negócio

---

### 6. Preparação para saída

- estruturar resultados
- garantir clareza
- preparar para visualização ou decisão

---

## 🔄 6. INTEGRAÇÃO COM OUTRAS SKILLS

A skill deve ser combinada conforme necessidade:

---

### 🔗 Combinações comuns

- data_analysis + reasoning → interpretação estruturada
- data_analysis + code_generation → análise automatizada
- data_analysis + validation → verificação de consistência
- data_analysis + clarification → levantamento de requisitos

---

## ✅ 7. VALIDAÇÃO INTERNA

A skill deve validar:

- os dados fazem sentido?
- os cálculos estão corretos?
- há consistência nos resultados?
- o insight tem base nos dados?

---

## ⚠️ 8. DETECÇÃO DE RISCO

A skill deve identificar:

- dados incompletos ou inconsistentes
- falta de contexto de negócio
- métricas mal definidas
- interpretação ambígua

---

### Ação obrigatória:

- sinalizar risco claramente
- evitar conclusões definitivas
- solicitar mais contexto quando necessário

---

## 🧾 9. PADRÕES DE QUALIDADE

A análise deve:

- ser clara e objetiva
- evitar ambiguidade
- ser baseada em dados reais
- manter coerência com o problema

---

### ✅ Boas práticas obrigatórias

- identificar claramente as métricas utilizadas
- explicar origem dos dados
- evitar interpretações sem evidência
- separar dados de opinião

---

## 🚫 10. PROIBIÇÕES

A skill não deve:

- inventar dados
- assumir números não fornecidos
- gerar insights sem base
- ignorar inconsistências
- confundir correlação com causalidade

---

## 🎯 11. CRITÉRIO DE QUALIDADE

A execução é considerada correta quando:

- a análise é consistente
- os resultados são confiáveis
- os insights são fundamentados
- a resposta é útil para decisão
